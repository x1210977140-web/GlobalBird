# 家里电脑同步数据包约定

## 1. 目的

这份文档定义“家里电脑 -> 国内 ECS”的最小数据交付格式。

目标是：

- 让家里电脑只负责采集和整理
- 让 ECS 只负责接收、导入和服务
- 让 worker 可以长期稳定支持“本地发布包导入”

## 2. 基本原则

- 数据包必须是版本化的，不允许覆盖式上传当前版本。
- 数据包必须可校验，至少提供文件 hash。
- 数据包必须能离线导入，不能要求 worker 在导入时再访问家里电脑。
- 原始来源信息必须保留，便于审计、追溯和重新导入。

## 3. 推荐目录结构

推荐每次发布一个目录：

```text
release-YYYYMMDD-HHMM/
  manifest.json
  birds.sqlite
  source/
  media/
```

说明：

- `manifest.json`
  记录版本元数据、来源、文件清单、hash、记录数。
- `birds.sqlite`
  当前阶段推荐的小规模静态快照主文件。
- `source/`
  存原始来源文件，例如 `dwca.zip`、`occurrence.txt`、`multimedia.txt`。
- `media/`
  可选。存允许同步的本地静态媒体资源。

如果当前阶段不想维护 `SQLite`，也可以改成：

```text
release-YYYYMMDD-HHMM/
  manifest.json
  dwca.zip
  source/
```

但从导入复杂度看，`SQLite` 快照通常更容易稳定落地。

## 4. manifest.json 最小字段

建议最小结构：

```json
{
  "version": "2026-04-03-2200",
  "createdAt": "2026-04-03T22:00:00+08:00",
  "sourceType": "sqlite-snapshot",
  "description": "bird snapshot for ecs import",
  "files": [
    {
      "path": "birds.sqlite",
      "sha256": "..."
    }
  ],
  "stats": {
    "speciesCount": 0,
    "occurrenceCount": 0,
    "mediaCount": 0
  }
}
```

如果来源是 GBIF，再加：

```json
{
  "gbif": {
    "downloadKey": "...",
    "doi": "...",
    "predicatePath": "source/predicate.json"
  }
}
```

## 5. ECS 目录约定

ECS 上建议使用以下目录：

```text
/opt/globalbird/data/inbox
/opt/globalbird/data/archive
/opt/globalbird/data/releases
```

语义如下：

- `inbox`
  家里电脑刚上传的待导入发布包。
- `archive`
  已成功导入的历史包。
- `releases`
  当前保留的有效版本快照。

## 6. 同步流程

推荐流程：

1. 家里电脑生成 `release-YYYYMMDD-HHMM/`
2. 在家里电脑本地校验 hash
3. 用 `rsync` 或 `scp` 上传到 ECS 的 `inbox`
4. ECS 上执行导入脚本
5. 导入成功后，将该版本移动到 `archive`
6. 需要长期保留时，再复制一份到 `releases`

## 7. 推荐同步方式

优先级：

1. `rsync + SSH`
2. `scp`
3. `sftp`
4. `Tailscale SSH`

当前阶段不建议：

- 让 ECS 运行时实时读取家里电脑文件
- 把家里电脑暴露成公网源站
- 依赖任何“必须在线保持连接”的家庭网络回源方式

## 8. worker 导入入口约定

worker 第一版建议支持两个入口：

- `import:release`
  从 `birds.sqlite` 之类的静态快照导入。
- `import:source`
  从 `dwca.zip` 或原始文本文件导入。

这样可以先用简单快照把链路跑通，后面再补原始导入。

## 9. 版本清理建议

- `inbox` 只保留待导入包。
- `archive` 至少保留最近 5 个成功版本。
- `releases` 只保留当前有效版本和上一个回滚版本。

## 10. 当前建议

当前阶段优先采用：

- 家里电脑生成 `SQLite` 快照
- 同步到 ECS
- worker 从本地文件导入 Postgres

不要先做：

- 在线 GBIF 自动抓取
- 家庭网络实时回源
- 复杂对象存储分发
